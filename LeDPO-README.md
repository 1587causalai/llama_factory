# LEDPO (可学习Beta DPO) 渐进式开发计划

本文档描述了从标准DPO开始，渐进式开发LEDPO功能的计划。每一步都会有明确的目标、验证标准和回退策略。

## 项目背景

LEDPO (可学习Beta DPO) 是对标准DPO算法的扩展，允许模型自动学习每个训练样本的最优beta值，理论上能够提高模型对人类偏好的学习效果。然而，在实践中遇到了beta值趋零的问题，我们需要通过系统的分析和逐步开发来解决这个问题。

## 开发阶段

### 阶段1: 标准DPO基准测试

**目标:** 建立标准DPO训练的基准性能

**步骤:**
1. 创建DPO基准配置文件 (`ledpo_progressive_dev/qwen15_lora_foodpo.yaml`)
2. 运行标准DPO训练
3. 记录和分析训练结果


```bash
llamafactory-cli train ledpo_progressive_dev/qwen15_lora_foodpo.yaml
```

**验证标准:** 
- 训练能够正常完成
- 损失函数正常下降
- 模型性能有合理提升

### 阶段2: 最小化LEDPO实现

**目标:** 实现最基本的LEDPO功能，包括可学习的beta值

**步骤:**
1. 复制DPO的trainer代码到新的LEDPO模块
2. 添加简单的ValueHead网络
3. 修改loss计算，引入动态beta
4. 创建LEDPO基本配置文件
5. 创建运行脚本
6. 运行并分析结果

**验证标准:**
- 训练能够正常完成
- ValueHead能够学习并输出合理的beta值
- beta值不会出现趋零问题

### 阶段3: 添加诊断和监控功能

**目标:** 增强LEDPO的监控和诊断能力，便于分析beta值变化

**步骤:**
1. 添加beta值记录功能
2. 实现beta趋势绘图函数
3. 添加训练过程中的beta分析
4. 创建训练后分析脚本

**验证标准:**
- 能够正确记录beta值变化
- 图表能够清晰展示beta趋势
- 分析报告能提供有价值的信息

### 阶段4: 解决beta趋零问题

**目标:** 彻底解决beta趋零问题，确保LEDPO算法稳定有效

**步骤:**
1. 实现不同的ValueHead变体
2. 添加beta正则化项
3. 优化参数初始化和学习率
4. 实验不同的freeze策略
5. 比较不同方案的效果

**验证标准:**
- beta值保持在合理范围
- 正delta对应的beta明显大于负delta对应的beta
- 训练稳定且效果良好

### 阶段5: 性能优化和稳定性测试

**目标:** 优化LEDPO的性能并确保其在不同设置下的稳定性

**步骤:**
1. 针对较大模型进行优化
2. 测试不同学习率和批次大小的效果
3. 实现长时间训练的监控和保护机制
4. 创建全面的评估脚本

**验证标准:**
- 训练效率有所提升
- 算法在不同设置下都能保持稳定
- 最终模型性能优于标准DPO

## 开发原则

1. **渐进式开发:** 每一步都基于前一步的稳定版本，确保变更可控
2. **充分测试:** 每个新功能都要经过充分测试，确保其稳定性
3. **保持简洁:** 代码应该简洁清晰，便于理解和维护
4. **详细记录:** 记录每一步的变化、结果和发现
5. **可回退:** 每一步都要有回退策略，以防出现意外问题
